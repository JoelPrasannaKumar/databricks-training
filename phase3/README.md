# Phase 3: Final ETL & Pipeline Practice Pack

## Objective
This phase transitions from writing isolated queries to thinking like a data engineer by following the ETL (Extract, Transform, Load) workflow. It focuses heavily on data ingestion, cleaning, transformation, and complete pipeline building.

## Contents
- **`samples/`**: Contains `customers.csv` and `sales.csv` tailored to practice missing values and invalid records.
- **`pipeline.sql`**: Contains structured SQL queries mimicking business logic.
- **`pipeline.py`**: The primary executable PySpark pipeline that performs the end-to-end ETL process.
- **`outputs/`**: Final reporting tables generated by the pipeline.

## Key Concepts Practiced
* **Extract:** Reading CSV files, inspecting schemas.
* **Transform:** Identifying missing values, dropping/filling nulls (`dropna()`, `fillna()`), filtering invalid records (`filter()`), and computing business metrics (daily sales, city-wise revenue).
* **Load:** Exporting resulting transformations to structured output files.
